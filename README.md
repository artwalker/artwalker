## Culprit

Engineering teams often investigate the same kinds of incidents again and again.

The useful knowledge already exists, but it is scattered across:

- Jira issue summaries
- descriptions and comments
- investigation notes
- attached logs and archives
- screenshots and evidence files
- historical incidents
- tribal knowledge inside senior engineers' heads

**Culprit turns this messy incident context into structured RCA knowledge.**

It helps teams generate:

- incident summaries
- symptom analysis
- probable root causes
- solution and mitigation suggestions
- technical insights
- evidence summaries
- follow-up action items
- reviewable RCA candidates
- reusable knowledge cards

The goal is simple:

**help engineering teams investigate incidents faster, preserve expert knowledge, reduce repeated failures, and make root-cause analysis more inspectable.**

## Current Status

Culprit is not just a concept or demo.

An earlier Jira RCA workflow has already been run successfully in a real company environment using a Hermes / OpenClaw-based implementation.

That validation showed a clear need:

Engineering teams do not just need another ticket summarizer.

They need a system that can:

- read the incident context
- collect evidence
- reason over symptoms and technical signals
- propose root-cause candidates
- preserve the evidence trail
- let humans review the conclusion
- promote reviewed findings into reusable knowledge

Every company uses Jira differently.

Different teams have different:

- issue types
- custom fields
- module names
- severity rules
- incident categories
- attachment formats
- RCA habits
- review processes
- knowledge-management workflows

Because of that, Culprit is being built as a workflow-specific incident intelligence product, not a generic Jira summarization tool.

It supports customization around:

- Jira field mapping
- incident workflow mapping
- evidence fetching rules
- RCA output structure
- review and promotion flow
- knowledge-card taxonomy
- team-specific incident vocabulary
- private deployment requirements

I am currently looking for pilot teams, early customers, technical partners, and investors who are interested in AI agents for incident analysis, RCA automation, and engineering knowledge systems.

## Sonechka

I am building **Sonechka**, my self-developed AI agent runtime, to productize and scale the workflow intelligence behind Culprit.

Sonechka is not a thin LLM wrapper.

It is designed for:

- reliable long-running task execution
- retrieval and evidence handling
- memory and continuity
- workflow automation
- auditability and traceability
- visible tool-use boundaries
- human review gates
- TUI and Telegram interaction surfaces
- multi-provider model support

The earlier Jira RCA workflow was validated with Hermes / OpenClaw.

Sonechka is the next-generation proprietary runtime I am building to make this kind of AI automation more reliable, inspectable, customizable, and commercially scalable.

The principle is:

**evidence before conclusion, review before promotion, runtime before model dependency.**
